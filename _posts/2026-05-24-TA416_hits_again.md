# TA416 hits Europe again 
From mid-2025 onwards, the China-aligned threat actor TA416 resumed observed targeting of European government and diplomatic organizations following a period of reduced EU-focused Attcaks .This Report focuses on attack chain  variant that utilizes MSbuild with Malicious C# project as initial dropper ,dropping Legitimate Gdata Antivirus ,wich will sideload the Plugx malware loader .This was first seen in the wild since February (<b>Although other infection chain variants was seen before ,like abusing Microsoft OAuth for auto redirection to attacker controlled site .
&nbsp;Note:For a trace of every seen delivery method see : [Proofpoint Threat research article ](https://www.proofpoint.com/us/blog/threat-insight/id-come-running-back-eu-again-ta416-resumes-european-government-espionage)
)</b><br>
The adversaries used various delivery methods ,but the End goal is the same ,running a shellcode reflective loader that will run Plugx backdoor in memory.



## Executive Summary 
**This is a plugx backdoor malware , A notorious very obfuscated backdoor ,which has been observed increasing in activity recently ,targeting EU diplomates and NATO delegates.
It can collect host information like OS info ,username/hostname ,All interfaces ip addresses (Which All could help shape follow on behaviors and malware delivery) ,then it will start a backdoor ,recieving commands from the C2 server via HTTPS ,Providing functionality like :Reverse shell ,Dropping More malware ,Cleaning up current malware setup**.

## Plugx Backdoor analysis

Malware stores its config in .rdata section as follows:

[keylength|RC4 key |Encrypted config]

![title](resources/config.png)


```
// Config stored :KEy||encrypted data
    Rc4AlgoWithKeyExpansion((char *)&EncryptedConfigBuffer, 0x8A0u, (int)&EncryptedConfigBuffer, Rc4Key);
```
The data is first decrypted with RC4 ,then decoded with the following algorithm :

```c
char *encoded=Rc4AlgoWithKeyExpansion((char *)&EncryptedConfigBuffer, 0x8A0u, (int)&EncryptedConfigBuffer, Rc4Key);
for(int i=0;i<strlen(encoded);i++){
  char temp1=(~i&0x18)&0xe7 |(i);
    char temp2=(~encoded[i]&0x18)&0xe7 |(encoded[i]);
    encoded[i]=temp1^temp2;
}
```
The malware decrypted :%public%\GDatas (<b>the malware config directory</b>),Mutex name (<b>config dependent</b>) ,and arp-news( <b>which will be sent with initial beacon</b>)
<br><br>

## Techniques , Tactics and procedures 

A.**Initial Access :**

phishing ,spearphishing link (T1566.002):
The adversaries targeted NATO Deplomats with phishing emails ,convincing them to download and execute the An executable (invitationletter.exe) that will show some form of meeting schedule




B.**Execution :**

Adversaries relied on <b>User Execution</b> to launch MSBuild (<b>Trusted Developer Utilities Proxy Execution ,MSBuild T1127.001</b>) which will Build and run a Dropper C# project

Also ,They relied on Dll Side loading (<b>Hicjack Execution Flow ,dll T!574.001</b>) to launch the ShellCode loader.


Moreover,it used (<b>Native Api T1106</b>) To achieve funcionality like reading shellcode file ,and Injecting shellcode in the threadpool (RtlRegisterWait).

The backdoor can open a reverse shell and use (<b>Command and scripting interpreter T1059 </b>) To execute commands and send results to adversary.

C.**Persistence :**
For persistence the malware Set The user Registry Run Key ,which will launch the Gdata Antimalware on startup ,causing the same loading chain to occur over and over (<b>Boot or Logon Autostart Execution: Registry Run Keys / Startup Folder  T1547.001</b>)

```value :G Data  , value data:"C:\Users\Public\GDatas\Avk.exe" 569 743```



D.**Stealth :**

Adversaries Set the configuration directory as a hidden Directory "Gdata" under %public% <b>(Hide Artifacts ,Hidden files and directories T1564.001)</b>

<b>Hijack Execution Flow T1574.001</b> Was used To launch Avk.dll( stage1 loader ).

Upon Recieved Command ,the malware cleans its config directory and deletes Run key <b>(Indicator Removal ,File deletion T1070.004 and Clear Persistence T1070.009)</b>


Plugx Is hidden encoded inside the Avktray.Dat ,which decodes and reflectively loads it .Also all of the malware config is encrypted with RC4 <b>(Obfuscated files or information ,Embedded poayload  ,Encrypted/encoded file T1027)</b>


The final Loader Relectively loaded Plugx backdoor ,and Calling its dllMain <b>( Reflective Code loading T1620 )</b> Achieving a full In Memory execution ,bypassing detections .

Using MSBuild <b>(Trusted Developer Utilities Proxy Execution ,MSBuild T1127.001)</b> As a signed ,frequently Used binary ,to evade detections.


The adversaries also create a Mutex with name extracted from the config ,To make sure only one malware instance is running at a time <b>(Execution Gaurdrails ,Mutual Exclusion T1480.002)</b>


E.**Discovery :**

The malware Enumerated Files in %appdata%\Render ,looking for certain files <b>(file and directory discovery T1083)</b>

Malware retrieves currently logged in user (<b>System owner discovery T1033</b>) as well as OS information ,like version ,hotfixes (<b>System Information Discovery T1082 </b>)


F.**Command and control :**

The adversaries used  HTTPS to send beacons and exfiltrate harvested data and command execution results (<b>Application Layer protocol ,web protocol T1071.001 </b>) ,this makes network inspection harder ,In addition ,Even the POSTed data is encrypted using RC4 (<b>Encrypted Channel ,Symmetric cryptography T1573.001 </b>) 

Upon A certain Command sent by the attacker ,the malware can download a new (EXE ,DLL,Dat) infection chain ,if the target is of interest (Ingress Tooll Transfer T1105)
